Les relations dans les bases de données se divisent en trois types principaux. La relation 1:1 (One-to-One) signifie que chaque enregistrement d’une table est lié à un seul enregistrement dans une autre table, et inversement, comme la relation entre User et Student (chaque étudiant possède un seul compte). La relation 1:N (One-to-Many) signifie qu’un enregistrement peut être lié à plusieurs enregistrements dans une autre table, mais chaque enregistrement de cette dernière ne dépend que d’un seul, comme une Class qui contient plusieurs Students. Enfin, la relation N:N (Many-to-Many) signifie que plusieurs enregistrements des deux côtés peuvent être liés entre eux, comme Students et Courses (un étudiant peut suivre plusieurs cours et un cours peut contenir plusieurs étudiants). Ce type de relation ne se fait pas directement, mais nécessite une table intermédiaire (comme Enrollments) pour l’organiser.
